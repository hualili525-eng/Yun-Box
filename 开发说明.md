# Yun Box 开发与维护说明

> 基于 Tab Out 二次开发的 Chrome 新标签页扩展，加入了自定义书签分组、Google 搜索框、自定义背景图等功能。

---

## 一、项目结构

```
tab-out-main/
├── extension/              ← Chrome 扩展实际加载的目录
│   ├── manifest.json       ← 扩展配置（名字、版本、权限）
│   ├── index.html          ← 新标签页结构
│   ├── style.css           ← 样式
│   ├── app.js              ← 全部业务逻辑
│   ├── background.js       ← 后台 service worker
│   └── icons/              ← 扩展图标
├── README.md
├── 开发说明.md              ← 本文件
└── .git/                   ← 本地 git 仓库
```

**关键点**：Chrome 加载扩展时选的是 `extension/` 这一层，不是外层目录。

## 二、本地开发流程

### 第一次安装

1. 打开 Chrome，地址栏输入 `chrome://extensions/`
2. 右上角开启「开发者模式」
3. 点击「加载已解压的扩展程序」
4. 选择 `/Users/huaerxi/Desktop/张咋啦/tab-out-main/tab-out-main/extension`

### 修改代码后

1. 改完 HTML/CSS/JS 后，回到 `chrome://extensions/`
2. 找到 Yun Box（Tab Out）卡片
3. 点击卡片右下角的**圆形刷新箭头**
4. 打开新标签页测试

> 不刷新扩展，新代码不会生效。

### 调试

- 在新标签页按 `F12` 打开控制台
- 看 Console 里的报错和 `console.log` 输出
- 改完代码记得刷新扩展

## 四、常见问题排查

### 问题 1：扩展显示「错误」按钮

**症状**：扩展卡片上有红色「错误」按钮。

**排查**：
1. 点「错误」按钮看具体报错
2. 常见错误：
   - `Content Security Policy` → HTML 里有内联事件（`onclick="..."`、`onerror="..."`），删掉换成 addEventListener
   - `Cannot read property of undefined` → JS 引用了不存在的 DOM 元素
3. 改完代码 → 刷新扩展 → 进「错误」页点「全部清除」

### 问题 2：扩展 ID 变了，数据丢失

**症状**：新加载扩展后，分组、网站全没了。

**原因**：开发者模式扩展的 ID 与文件路径绑定。文件夹移动、改名都可能导致 ID 变化。新 ID 读不到旧 ID 的数据。

**数据恢复**：
旧数据存放在：
```
~/Library/Application Support/Google/Chrome/Profile 2/Local Extension Settings/<旧ID>/
```

可以把旧 ID 文件夹的内容复制到新 ID 文件夹下（**操作前先关闭 Chrome**）。

**预防**：固定文件夹位置不要随便移动。

### 问题 3：推送 GitHub 报 `Authentication failed`

**原因**：Token 失效或没配置。

**解决**：重新生成 Token，按"配置免密推送"步骤操作。

### 问题 4：本地文件夹删了，扩展失效

**预期行为**。开发者模式扩展依赖本地文件，删了就坏。

**预防**：
- 文件夹不要放在容易误删的地方（比如桌面）
- 推荐位置：`~/Documents/yun-box`
- 重要改动及时 push 到 GitHub
- 想彻底脱离本地文件 → 发布到 Chrome Web Store

### 问题 5：扩展从扩展列表消失

**可能原因**：
- 开发者模式被关闭（看右上角开关）
- Chrome 更新后自动禁用了未签名扩展
- 文件夹被移动/删除

**解决**：重新点「加载已解压的扩展程序」，选择 extension 目录。

## 五、数据存储说明

### 用户数据存放在哪？

| 数据 | 存储位置 | 说明 |
|------|---------|------|
| 自定义书签分组 | `chrome.storage.local` (`customBookmarks`) | 每个分组的名称和网站列表 |
| 背景图片 | IndexedDB (`tabout-bg`) | 原始图片 Blob，不限大小 |
| Saved for Later | `chrome.storage.local` (`deferred`) | Tab Out 原有功能 |

**注意**：这些数据存在 Chrome 浏览器里，跟扩展 ID 绑定，**不在项目文件夹里**。删除文件夹不会丢数据，但更换扩展 ID 会读不到。

### 备份用户数据

在新标签页按 F12 打开控制台，运行：

```js
// 导出书签
chrome.storage.local.get('customBookmarks', console.log)
// 然后复制控制台输出的 JSON 内容
```

恢复：

```js
chrome.storage.local.set({ customBookmarks: 这里粘贴JSON })
```

## 六、发布到 Chrome Web Store（可选）

发布后无需依赖本地文件，跨设备自动同步。

### 准备工作

1. 确认 `manifest.json` 里的 `version` 已升级
2. 确认 `icons/` 目录有 16/48/128 三个尺寸的图标
3. 准备截图（至少 1280×800 或 640×400）
4. 准备简介文字（中英文）

### 打包

把 `extension/` 文件夹（不是外层）压缩成 zip：

```bash
cd "/Users/huaerxi/Desktop/张咋啦/tab-out-main/tab-out-main"
zip -r yun-box.zip extension
```

### 提交

1. 访问 https://chrome.google.com/webstore/devconsole
2. 注册开发者账号（一次性 $5）
3. 上传 zip，填写描述、隐私政策、截图
4. 提交审核（1-3 个工作日）
